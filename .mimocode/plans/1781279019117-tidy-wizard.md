# iconfont字体图标支持实现计划

## 需求分析
实现iconfont字体图标支持，允许用户在文档中使用iconfont字体图标。

## 实现方案

### 1. 创建iconfont插件
**文件**: `src/plugins/iconfont.ts`

插件功能：
- 提供内置的iconfont图标库（包含log和code图标）
- 支持用户自定义iconfont资源（通过配置）
- 在`init`生命周期中动态加载iconfont CSS
- 提供图标组件渲染方法

```typescript
export default {
    init() {
        // 加载内置iconfont CSS
        // 支持用户自定义iconfont URL
    },
    // 提供图标渲染工具方法
    renderIcon(name: string, options?: { color?: string; size?: string }): string
}
```

### 2. 修改webpack配置
**文件**: `build/webpack.base.config.js`

添加字体文件loader：
```javascript
{
    test: /\.(woff|woff2|eot|ttf|otf)$/,
    type: 'asset/resource'
}
```

### 3. 创建iconfont资源
**目录**: `ctx/static/iconfont/`

包含：
- `iconfont.css` - 图标样式定义
- `iconfont.woff` / `iconfont.woff2` / `iconfont.ttf` - 字体文件

### 4. 注册插件到内置主题
**文件**: 
- `src/plugins/default/index.ts`
- `src/plugins/simple/index.ts`

将iconfont插件添加到插件列表中。

### 5. 更新配置支持
**文件**: `src/config/config.ts`

添加配置项：
```typescript
// iconfont配置
iconfont: {
    url: '', // 自定义iconfont CSS URL
    prefix: 'iconfont' // 图标类名前缀
}
```

### 6. 更新文档示例
**文件**: `ctx/docs/demo/demo.md`

确保现有的iconfont示例能够正常工作。

## 关键文件清单
- `src/plugins/iconfont.ts` (新建)
- `build/webpack.base.config.js` (修改)
- `ctx/static/iconfont/` (新建目录)
- `src/plugins/default/index.ts` (修改)
- `src/plugins/simple/index.ts` (修改)
- `src/config/config.ts` (修改)

## 验证方案
1. 运行 `yarn build` 确保构建成功
2. 运行 `yarn start` 启动开发服务器
3. 访问 demo 页面，验证iconfont图标正常显示
4. 测试自定义iconfont配置是否生效
