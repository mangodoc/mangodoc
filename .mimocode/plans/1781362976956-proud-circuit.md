## 四、推荐实施计划（按优先级排序）

### Phase 1: Bug 修复 + 基础加固（高优先级）

**1.1 修复 Store 单例 bug**
- 文件：`src/store/base.ts`, `src/store/store.ts`, `src/store/flag.ts`, `src/store/local.ts`, `src/store/nav.ts`
- 方案：每个子类覆写 `getInstance()` 创建独立实例，或重构 Base 不预创建实例

**1.2 fetch.ts 添加错误处理**
- 文件：`src/util/fetch.ts`
- 方案：添加 `.catch()` 处理，隐藏 loading 并显示错误提示（如 "页面加载失败"）

**1.3 callHook 添加 try/catch**
- 文件：`src/util/util.ts`
- 方案：每个插件调用包裹 try/catch，异常时 console.warn 并继续执行下一个插件

**1.4 修复 copycode 复制失败反馈**
- 文件：`src/plugins/copycode.ts`
- 方案：使用 `navigator.clipboard.writeText()` API，捕获异常显示"复制失败"

**1.5 修复 getConfigOrDefault falsy 检查**
- 文件：`src/util/util.ts`
- 方案：`window.$mangodoc[key] !== undefined ? window.$mangodoc[key] : config[key]`

### Phase 2: 中优先级新特性

**2.1 Mermaid 图表插件**
- 新文件：`src/plugins/mermaid.ts`
- 方案：在 `afterEach` 钩子中检测 ` ```mermaid ` 代码块，动态加载 mermaid CDN，调用 `mermaid.run()` 渲染
- 注册到三款主题

**2.2 目录高亮追踪**
- 新文件：`src/plugins/cool/scrolltrack.ts`（仅 cool 主题）
- 方案：在 `doneEach` 中监听 scroll 事件，使用 IntersectionObserver 或 getBoundingClientRect 计算当前章节，高亮侧栏对应菜单项
- 需要与 aside 插件配合，给菜单项添加 data-heading 属性

**2.3 Markdown 容器语法**
- 新文件：`src/plugins/container.ts`
- 方案：在 `beforeEach` 钩子中用正则解析 `::: tip/warning/danger` 语法，替换为带样式的 HTML div
- 在三款主题的 css.ts 中添加对应样式

### Phase 3: 低优先级特性

**3.1 打印/PDF 导出**
- 新文件：`src/plugins/print.ts`
- 方案：添加 `@media print` 样式隐藏导航/侧栏/页脚，添加打印按钮

**3.2 SEO 元信息**
- 新文件：`src/plugins/seo.ts`
- 方案：在 `afterEach` 中从 markdown 提取标题，动态设置 `document.title`、`meta description`、OG 标签

**3.3 国际化 i18n**
- 新文件：`src/i18n.ts`
- 方案：提供语言包机制，导航栏支持语言切换，markdown 路径支持语言前缀

## 五、验证方案

每个 Phase 完成后：
1. `npm run build` 确认编译通过
2. 用 Puppeteer 在 `http://127.0.0.1:8081` 上验证功能
3. 手动测试：深色/浅色模式切换、插件异常时不白屏、网络断开时显示错误
4. 提交 git commit 并推送

## 六、建议优先级

**先修 Bug 再加特性**。Phase 1 的 5 个 bug 修复工作量小（预计 1-2 小时），但对稳定性和用户体验影响大。Phase 2 的 3 个特性是用户最可能需要的（Mermaid、目录追踪、容器语法），也是 README 中标记为中优先级的项目。
